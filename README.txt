Paywall APP

Архитектура / State managment:
Я выбрал Clean Architecture с MVVM и RiverPod как оптимальный стек для 2026 года.
Это сочетание дает четкое разделение ответственности, и готовность к масштабированию.

Так как проект небольшой это решение является самым удобным, с проектами больших масштабов
я бы предпочёл Bloc и активное использование Freezed для строгой типизации состояний.


UI Kit:
Для упрощения читаемости кода, оптимизации работы приложения (за счёт const конструкторов)
и минимизащии лишних импортов, написал простой UI Kit.

Он состоит из 3 основных разделв:
Essential (виджеты которые используются в приложении постоянно)
Frequent (Виджеты которые используются 3-5 раз)
Special (Виджеты которые чаще всего используются единожды и сделаны под определённую страницу)


БД:
В данно проекте выбрал Shared Preferences исходя из ТЗ.
В больших проектах, особенно где есть оплаты или переводы предпочёл бы выбор РБД, Например SQL Lite.


Кодогенерация:
В данном проекте использована простая генерация путей к svg иконкам.


Навигация:
Go Router


Использование Нейросетей:
Для этого проекта использовал Grok / Deep Seek и Promt Canon.

В основном нейросети использовал для быстрой генерации:
 - абстракций
 - файлов с константами
 - именнованных конструкторов для виджетов в Ui Kit
 - написания простых виджетов / провайдеров

Promt Canon (даёт сразу ответ 12 нейросетей) использовал для:
 - выбора архитектуры проекта
 - общего анализа некоторых файлов на предмет улучшений с точки зрения кода / оптимизации
 - написания промтов для Deep Seek / Grok
